Images

<img>

Descriptions:
The <img> tag defines an image in an HTML page.
The <img> tag has two required attributes: src and alt.
Notes:
Images are not technically inserted into an HTML page, images are linked to HTML pages. The <img> tag creates a holding space for the referenced image.
To link an image to another document, simply nest the <img> tag inside <a> tags.

align

Descriptions:
The align attribute specifies the alignment of an image according to the surrounding element.
The <img> element is an inline element (it does not insert a new line on a page), meaning that text and other elements can wrap around it. Therefore, it can be useful to specify the alignment of the image according to surrounding elements.
Notes:
The align attribute of <img> is not supported in HTML5. Use CSS instead.

Attributes

ValueDescription
left Align the image to the left.
right Align the image to the right.
middle Align the image to the middle.
top Align the image to the top.
bottom Align the image to the bottom.

alt

Descriptions:
The required alt attribute specifies an alternate text for an image, if the image cannot be displayed.
The alt attribute provides alternative information for an image if a user for some reason cannot view it (because of slow connection, an error in the src attribute, or if the user uses a screen reader).
Notes:
To create a tooltip for an image, use the title attribute!
Internet Explorer 7 (and earlier) displays the value of the alt attribute as a tooltip, when mousing over the image. This is NOT the correct behavior, according to the HTML specification.

Attributes

ValueDescription
text Specifies an alternate text for an image.
Guidelines for the alt text:
  • The text should describe the image if the image contains information.
  • The text should explain where the link goes if the image is inside an <a> element.
  • Use alt="" if the image is only for decoration

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="">

</div>

crossorigin

Descriptions:
Allow images from third-party sites that allow cross-origin access to be used with canvas.

Attributes

ValueDescription
anonimous Before downloading the execution script, cross-domain request (Origin: HTTP header), while not passing access parameters (such as: cookie, X.509 certificate, login / password for basic HTTP authentication). The server response must contain a header. Access-Control-Allow-Origin: HTTP header.
use-credentials Before downloading the script, a cross-domain request (Origin: HTTP header) is executed with the indication of access parameters (in the form of a cookie, a certificate or a login / password pair). The server response must include an Access-Control-Allow-Origin: HTTP header, otherwise, the use of the image on <canvas> is restricted. If this attribute is not specified, then CORS is not used when loading the script (there is no Origin: HTTP header).

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="" crossorigin="use-credentials">

</div>

height

Descriptions:
The height attribute specifies the height of an image, in pixels.
Notes:
Always specify both the height and width attributes for images. If height and width are set, the space required for the image is reserved when the page is loaded. However, without these attributes, the browser does not know the size of the image, and cannot reserve the appropriate space to it. The effect will be that the page layout will change during loading (while the images load).
Downsizing a large image with the height and width attributes forces a user to download the large image (even if it looks small on the page). To avoid this, rescale the image with a program before using it on a page.

Attributes

ValueDescription
pixels The height in pixels.

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="" height="" width="100">

</div>

ismap

Descriptions:
The ismap attribute is a boolean attribute.
When present, it specifies that the image is part of a server-side image-map (an image-map is an image with clickable areas).
When clicking on a server-side image-map, the click coordinates are sent to the server as a URL query string.
Notes:
The ismap attribute is allowed only if the <img> element is a descendant of an <a> element with a valid href attribute.

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="" height="120" width="120" ismap>

</div>

longdesc

Descriptions:
The longdesc attribute specifies a hyperlink to a detailed description of an image.
Notes:
Attrbut longdesc is not supported by any of the browsers.

Attributes

ValueDescription
text A hyperlink to a detailed description of an image.
Possible values:
  • An id to another element.
  • An absolute URL - points to another web site (like longdesc="http://www.example.com/description.txt").
  • A relative URL - points to a file within a web site (like longdesc="description.txt").

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="" height="120" width="120" longdesc="">

</div>

src

Descriptions:
The required src attribute specifies the URL of the image.
Notes:
When a web page loads; it is the browser, at that moment, that gets the image from a web server and inserts it into the page. Therefore, make sure that the image actually stay in the same spot in relation to the web page, otherwise your visitors will get a broken link icon. The broken link icon is shown if the browser cannot find the image.

Attributes

ValueDescription
URL The URL of the image.
Possible values:
  • An absolute URL - points to another web site (like src="http://www.example.com/image.gif").
  • A relative URL - points to a file within a web site (like src="image.gif").

Examples

<div>

<img src="..images/vasserman_thinking.gif" alt="" height="120" width="120">

</div>

usemap

Descriptions:
The usemap attribute specifies an image as a client-side image-map (an image-map is an image with clickable areas).
The usemap attribute is associated with a <map> element's name attribute, and creates a relationship between the <img> and the <map>.
Notes:
The usemap attribute cannot be used if the <img> element is a descendant of an <a> or <button> element.

Attributes

ValueDescription
#mapname A hash character ("#") plus the name of the <map> element to use.

Examples

<div>

<img src="..images/vasserman_big.gif" alt="" height="220" width="250" usemap="#vasserman">

<map name="vasserman">

<area shape="rect" coords="50, 150, 200, 240" alt="beard" href="https://en.wikipedia.org/wiki/Beard">

<area shape="circle" coords="100, 88, 25" alt="glasses" href="https://en.wikipedia.org/wiki/Glasses">

<area shape="circle" coords="150, 88, 25" alt="glasses" href="https://en.wikipedia.org/wiki/Glasses">

</map>
</div>
